# 实验1: 中级图元发现实验

## 动机
当前实验中，基础图元（线段、正方形等）通常是有用的构建块。但如果最有用的图元实际上是中级图元（如"角落"、"边框"、"十字中心"等），人们能否发现并优先使用这些中级图元？

## 实验设计

### 刺激集设计原则
1. **最短程序长度**: 使用中级图元比基础图元能产生更短的程序
2. **重复性**: 中级图元在多个目标模式中重复出现
3. **组合性**: 目标模式主要由中级图元的组合构成

### 候选中级图元
```javascript
// 角落图元 (Corner primitives)
corner_tl: () => // 左上角 L 形状
corner_tr: () => // 右上角 L 形状  
corner_bl: () => // 左下角 L 形状
corner_br: () => // 右下角 L 形状

// 边框图元 (Border primitives)  
border_top: () => // 顶部边框
border_bottom: () => // 底部边框
border_left: () => // 左侧边框
border_right: () => // 右侧边框

// 十字组件 (Cross components)
cross_center: () => // 十字中心 (2x2 方块)
cross_arms: () => // 十字臂 (不包括中心)

// 对角元素 (Diagonal elements)
diag_quarter_1: () => // 对角线第一象限
diag_quarter_2: () => // 对角线第二象限
```

### 目标模式集
设计25个目标模式，其中：
- 20个模式的最短程序使用中级图元
- 5个模式作为对照，最短程序使用基础图元
- 中级图元在不同模式中的重用率 > 60%

### 实验流程
1. **阶段1**: 只提供基础图元，观察参与者如何构建模式
2. **阶段2**: 引入"库学习"机制，参与者可以保存自己创建的中级图元
3. **阶段3**: 测试阶段，看参与者是否会重用发现的中级图元

### 测量指标
- 中级图元发现率
- 中级图元重用频率  
- 程序长度变化
- 任务完成时间
- 主观有用性评分

## 预期结果
- 参与者会逐渐发现有用的中级图元
- 发现中级图元后，程序长度显著减少
- 个体差异：某些人更善于发现抽象的中级图元
