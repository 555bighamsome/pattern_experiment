# 实验2: 像素级构建实验 (Pixel-Level Construction)

## 动机
从最底层的像素开始，观察人们如何自发地创建组合图元。这将揭示人类在构建抽象概念时的归纳偏见，并可与无DSL的LLM库学习模型进行比较。

## 实验设计

### 基础设置
- **画布**: 10x10 像素网格
- **初始工具**: 只有单像素绘制/擦除功能
- **目标**: 重现与原实验相同的目标模式

### 库学习机制
```javascript
// 参与者可以创建和命名自己的工具
const userLibrary = {
    // 示例：参与者可能会创建
    "my_line": {pattern: [[1,1,1,1,1]], description: "水平线段"},
    "corner": {pattern: [[1,0],[1,1]], description: "角落"},
    "plus": {pattern: [[0,1,0],[1,1,1],[0,1,0]], description: "加号"}
};

// 工具可以参数化
const parameterizedTools = {
    "line": (length, orientation) => generateLine(length, orientation),
    "rectangle": (width, height, filled) => generateRectangle(width, height, filled)
};
```

### 实验流程
1. **训练阶段** (5个简单模式)
   - 学习基本交互
   - 发现可以创建和保存工具

2. **自由探索阶段** (10分钟)  
   - 给定一组代表性目标模式预览
   - 鼓励创建有用的工具
   - 不限制工具数量

3. **任务阶段** (20个目标模式)
   - 可以使用已创建的工具
   - 可以继续创建新工具
   - 记录所有交互

### 数据收集
- 创建的工具序列及其使用频率
- 工具的抽象层次（从简单形状到复杂组合）
- 参数化程度（固定形状 vs 可变参数）
- 语义命名模式

### 与LLM基线比较
- 使用相同的像素级设置训练LLM库学习模型
- 比较人类和模型发现的图元类型
- 分析抽象层次的差异

## 预期发现
- 人类会快速抽象出线段、角落等基础形状
- 会基于视觉相似性而非功能相似性进行分组
- 命名会反映功能或视觉特征
- 存在个体差异：有些人更倾向于创建高级抽象
